# XRPL Amendment Unknowns: Second-Pass Provenance Research

This is a second pass over amendments that were marked `Unknown` in `xrpl-amendments-10y-manual-review-table.md`.

Scope: XRPL known-amendments universe present after 2016-06-04. This table uses implementation provenance from the local `XRPLF/rippled` checkout plus the existing XLS provenance packet where available. Implementation provenance is not always the same thing as original idea authorship, but it is stronger than leaving a row as unknown when the amendment was introduced through a named rippled commit.

Classification rule:

- `Direct Ripple` means the evidence row itself has a `@ripple.com` email, or the same person is already tied to a direct Ripple email in the reviewed XLS/commit evidence.
- `XRP ecosystem / external` means the author is a named XRPL ecosystem contributor or outside contributor, but the researched evidence does not prove Ripple employment for that row.
- `Unresolved` means the commit identity is visible, but I am not asserting organizational status.

## Main Takeaway

The unknown bucket mostly collapsed into implementation provenance. The majority of the researched fixes are amendment-gated repairs to `rippled` itself: payment pathing, offer crossing, amount canonicalization, NFT page/link invariants, AMM rounding/accounting, MPT metadata, escrow/token behavior, bridge reward rounding, and amendment/validation mechanics. That is the answer to "bug fixes for what": bug fixes for the core XRPL server's transaction, ledger-entry, and feature-integration code paths.

The table also preserves exceptions. Some rows are clearly non-Ripple or external ecosystem implementation authors, and a few remain unresolved because the researched source only gives a GitHub identity or pseudonymous commit author.
